PetSOS

Aplicativo mobile para registrar, acompanhar e moderar denúncias relacionadas a animais. O projeto combina mapa, notificações, voluntariado, evidências, perfis e painel administrativo para dar visibilidade e organização aos casos.

Ver roadmap Ver frontend Ver backend

O que é

Uma plataforma de apoio comunitário para denúncias de abandono, maus-tratos, negligência e outras ocorrências envolvendo animais.

Para quem

Pessoas que querem reportar casos, acompanhar denúncias próximas, contribuir como voluntárias e validar resoluções com evidências.

Funções do aplicativo

Resumo funcional

Denúncias e mapa

  • Criação de denúncia com fotos, descrição, tipo, animal e localização.
  • Mapa com marcadores, popup de denúncia, detalhe e rota até o local.
  • Lista de denúncias com busca, filtros e cards otimizados.
  • Detalhe público para usuários logados e convidados.

Comunidade e conta

  • Cadastro, login, lembrar identificador e recuperação de senha.
  • Perfil com foto, nome, descrição, localização e denúncias acompanhadas.
  • Acompanhamento, comentários, respostas, reportes e notificações.
  • Voluntariado e envio de evidências para apoiar a resolução.

Arquitetura resumida

Camadas principais

App mobile

Expo Router organiza rotas e tabs. Hooks conectam estado de tela, cache local e serviços HTTP. O app usa Firebase Auth para identidade e consome a API Express.

API backend

Express recebe requests, valida payloads com Zod, aplica permissões, centraliza regras de negócio e acessa Firestore via Firebase Admin.

Dados e integrações

Firestore guarda usuários, denúncias, seguidores, notificações, comentários e moderação. O app também consome contratos de API para rotas e sugestões de cidade/UF.

Sistemas implementados

Performance, segurança e operação

Otimização

  • Cache local com stale-while-revalidate.
  • Endpoints agregados para reduzir round-trips.
  • Tiles de mapa com índice, batch e invalidação em tempo real.
  • Thumbnails com Sharp e cache de imagens no app.

Segurança

  • Backend como autoridade final das permissões.
  • Roles administrativas e proteção de rotas sensíveis.
  • Validação de input com Zod.
  • Chaves externas mantidas no backend.

Observabilidade

  • Logs estruturados com Pino.
  • Métricas Prometheus protegidas por token.
  • Swagger/OpenAPI para consulta das rotas.
  • Benchmarks com Autocannon.
