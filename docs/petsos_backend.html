Backend PetSOS API

API Node.js com Express 5, Firebase Admin e Firestore. O backend centraliza validação, permissões, uploads, rotas, cache de mapa, notificações, moderação e integrações externas.

Stack

Dependências principais
Node.jsExpress 5Firebase AdminFirestoreZodMulterSharpHelmetCORSPinoPino HTTPprom-clientSwagger UIgeofire-commonwsAutocannon

Domínios

  • Usuários, autenticação e perfil.
  • Denúncias, seguidores, voluntários e evidências.
  • Comentários, respostas, notificações e moderação.

Mapa

  • Geohash para consultas próximas.
  • Tiles indexados e endpoint batch.
  • Realtime de invalidação para atualizar marcadores.

Integrações

  • OpenRouteService para rotas por ruas.
  • Contrato /locations/cities consumido pelo app para sugestões de cidade/UF.
  • Uploads locais em /uploads com thumbnails.

Endpoints e fluxos importantes

Contratos usados pelo app

Performance

  • GET /api/app/bootstrap carrega dados mínimos de inicialização.
  • GET /api/users/me/summary agrega perfil, resumo e notificações.
  • GET /api/users/:username/profile-summary entrega perfil público agregado.
  • POST /api/complaints/map/tiles/batch reduz múltiplos GETs de tiles.

Produto

  • /api/complaints cobre listagem, criação, detalhe, edição e exclusão.
  • /api/notifications lista, marca como lida e limpa notificações.
  • /api/comments e rotas de denúncia suportam comentários e reportes.
  • /api/routes/driving faz proxy seguro para rotas do OpenRouteService.

Segurança

Controles implementados

Autorização

  • Firebase token validado no backend.
  • Admin como role para moderação e exclusões sensíveis.
  • Backend decide permissões; o frontend apenas reflete estado.

Entrada e abuso

  • Payloads validados com Zod.
  • Rate limit para reportes.
  • Email temporário bloqueado e domínio checado via MX.

Arquivos e segredos

  • Uploads com validação de tipo e tamanho.
  • Thumbnails processadas com Sharp.
  • Chaves externas ficam no backend, não no app.

Observabilidade e manutenção

Diagnóstico operacional

Logs e métricas

  • Pino e Pino HTTP registram requests, erros e eventos de domínio.
  • /api/metrics expõe métricas Prometheus com token.
  • Swagger/OpenAPI documenta rotas para inspeção manual.

Scripts

  • npm run backfill:complaint-geohashes popula geohashes antigos.
  • npm run backfill:map-tile-stats recalcula estatísticas dos tiles.
  • npm run bench mede rotas críticas com Autocannon.

Comandos

Desenvolvimento
  • npm install instala dependências.
  • npm run dev inicia a API com Nodemon.
  • npm start inicia em modo normal.
  • npm test executa testes unitários disponíveis.
  • npm run lint:check verifica formatação.
  • npm run bench executa benchmark controlado.
